# Screen Inventory

Phase 1 implements Splash, Login, adaptive application shell, Dashboard placeholder state, Profile, Settings, Access Denied, and general loading/error states. Navigation destinations are declared for Clients, Machines, Upcoming Services, Service Records, Book In, Job Cards, Knowledge Base, Calendar, Invoice Queue, and User Management, but feature screens are implemented only in their scheduled phases. Restricted destinations are never shown without their effective permission.

Compact uses bottom navigation plus overflow; medium uses a navigation rail; expanded uses a permanent drawer. Forms use single-pane compact layouts and constrained/two-pane expanded layouts in later phases.

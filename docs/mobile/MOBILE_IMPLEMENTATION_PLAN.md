# Mobile Implementation Plan

1. Foundation: Compose project, build variants, theme, Retrofit, encrypted session, authentication, permissions, adaptive shell.
2. Core operations: paginated clients/machines/services/job cards with list/detail/forms.
3. Offline/media: Room caches and drafts, UUID sync queue, WorkManager, conflict UI, compressed multipart attachments.
4. Commercial workflow: authoritative parts/labour totals, invoice drafts, signature, server PDF/email, truthful Sage adapter status.
5. Extended modules: Calendar, Knowledge Base, user administration, profile/settings.
6. Release: accessibility, adaptive UI tests, integration tests, performance, CI, signing documentation.

Each phase must pass its tests before the next begins. Laravel remains authoritative and all offline writes retain a visible sync state.
